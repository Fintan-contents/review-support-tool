## テスト実行設定の星取表

各シナリオの `config.yaml` から読み取った設定。○ = 有効/変更あり、× = デフォルト/変更なし。

設定項目の説明：
- **use_review_record**: レビュー記録票転記を使用するか（デフォルト: true）
- **use_summary**: レビュー記録サマリ（一覧ファイル）を使用するか（デフォルト: false）
- **review_times**: 抽出を複数回実行するか（○ = 複数回実行）
- **categories 変更**: xlsm デフォルト（9 カテゴリ）から変更するか
- **skip_open_files**: 一部 xlsx を Python から開かない設定があるか
- **named_ranges 変更**: 名前付き範囲の値を上書きするか
- **controls 変更**: チェックボックス等のコントロール値を変更するか
- **item_mapping_cells 変更**: 項目マッピング設定シートのセルを直接書き換えるか

## シナリオ詳細
### S01: 基本抽出（正常系）
> テスト種別: 自動
> レビューコメントを含む設計書と記録票を使い、基本的な指摘抽出・転記が正しく動作することを検証する。

**フィクスチャ**: `auto/scenario01/`

| ファイル | 内容 |
|---------|------|
| `システム機能設計書_サンプル_S01.xlsx` | A4=`*`（日時）, B5=`a`（未対応）, E60=`b`（対応中）, E62=`c`（クローズ）, A71=カテゴリなし（無効） |
| `システム機能設計書_サンプル_レビュー記録票_S01.xlsx` | 標準サンプル |

**設定**

| 設定項目 | 値 |
|---------|---|
| レビュー記録票使用 | TRUE |
| レビュー記録サマリ使用 | FALSE |
| 処理対象パターン | `.*機能設計書.*` |
| 処理対象外パターン | `.*レビュー記録票.*` |
| 指摘分類 | a, b, c（デフォルト） |
| REVIEW_TIMES（G4） | 1 |
| 開いているファイル | 設計書・記録票のみ |

**テスト手順**

| # | 操作 | 期待結果 | 観点 |

| 1 | 抽出ボタン押下 | 「抽出しますか？」確認ダイアログ表示 | 19 |
| 2 | 「はい」を選択 | 処理完了メッセージ表示 | - |
| 3 | 設計書の「レビュー結果1回目」シートを確認 | シートが生成されている | 09 |
| 4 | ヘッダー A4, C4, E4 を確認 | ファイル名・回数(1)・実施日(2023/08/01) が書き込まれている | 01, 10 |
| 5 | 7 行目 B〜D 列を確認 | a, b, c が書き込まれている | 11 |
